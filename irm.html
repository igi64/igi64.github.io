WebRTC and Identity Relationship Management

A work in progress.

Identity and Access Management (IAM) services were built for internal use behind the firewall. They were not designed to manage identity and access control across corporate boundaries. In the new cloud, social, and mobile era, many enterprises now find themselves with more identity silos than ever.

There is an initiative to design a new dynamic Identity Relationship Management (IRM) system that serves employees and customers, partners and devices.

The main idea of my approach is to replace Identity and Access Control with Identity-Based Connection (IBC), then on top of it build IRM system.

Today a new open standard emerges for real-time communication with potential to change the way we connect with businesses. WebRTC, a free open project authored by Google, now being drafted as an API definition by the W3C, allows for real-time, peer-to-peer video, audio, and data transfer between browsers.

Every single WebRTC browser client is de facto IoT node. It doesn't have a static ip address or domain identifier. In order for a WebRTC application to set up a connection, its nodes need to exchange some information. The signaling server is used to coordinate this communication.

The Identity-Based Privacy (IBP) trusted model offers a new way to manage meshed peer-to-peer communication between people, devices, and things.

IBP architecture is a better fit for the distributed nature of cloud computing.
